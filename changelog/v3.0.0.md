## Summary

Closes the error management loop. When a fix MR merges, Fido automatically resolves the error in Datadog. It then continuously monitors all tracked issues for status changes — if Datadog reopens a resolved issue, Fido detects the regression and surfaces it prominently in the dashboard.

## Highlights

- **Automatic Datadog resolution** — When an issue's MR status becomes `merged`, Fido calls the Datadog Error Tracking API (`PUT /issues/{id}/state`) to mark it `RESOLVED`. This happens once per fix cycle, with a `ResolvedAt` guard to prevent resolve loops.
- **Continuous status monitoring** — Every sync cycle, the engine checks Datadog's current status for all tracked non-ignored issues via `GET /issues/{id}`. Status divergence is detected and persisted.
- **Regression detection** — If a `resolved` issue flips back to `OPEN`/`FOR_REVIEW` in Datadog, Fido increments a regression counter, fires an `issue:regression` SSE event, and sends a browser notification.
- **Dashboard status badges** — New "DD Status" column with color-coded badges: green "Resolved", red "Regression (N)", muted for acknowledged/ignored. Regressions sort to the top.
- **SSE events** — Three new event types: `issue:resolved`, `issue:regression`, `issue:status_changed` — all handled by the frontend with highlight animations and notifications.
