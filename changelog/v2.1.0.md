## Summary

Enriches the investigation agent prompt with richer Datadog context so the agent can make better decisions about where to look and what kind of problem it's dealing with. Previously the agent only received the error report (stack trace, occurrence count, links). Now it gets trace payload details, error frequency patterns, version info, and co-occurring errors.

## Highlights

- **Trace details extraction** — Extracts error name, message, type, HTTP method/URL/status code, and response body from Datadog span payloads (beyond just the stack trace)
- **Error frequency buckets** — Daily occurrence timeseries via the Spans Aggregate API, with computed onset time, trend direction, and spike detection
- **Version info** — `FirstSeenVersion` / `LastSeenVersion` extracted from Datadog issue attributes and persisted in metadata through scan and import flows
- **Co-occurring errors** — Cross-references other tracked issues in the same service with overlapping first_seen timestamps (±1h tolerance)
- **Red herring guardrails** — Conditional inclusion rules (no empty sections) and framing text that discourages the agent from chasing false correlations
- **Prompt guidance** — Instructs the agent to correlate frequency onset with version changes using git log/blame

## Tested Against
