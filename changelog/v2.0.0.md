## Summary

Refactored the background scanner from a simple timer into a job-based sync engine with priority queue and token-bucket rate limiting. This is the foundation for the remaining Fido v3 features: resolution lifecycle, dashboard enrichment, agent investigation enrichment, notification service, and config UI.

## Highlights

- Priority job queue (heap-based) with four job types: `sync_issues`, `fetch_buckets`, `fetch_stacktrace`, `resolve_check`
- Token-bucket rate limiter for Datadog API calls (configurable, default 30/min)
- Sync engine worker drains queue at rate limit, spreading API calls across scan interval
- Pre-fetches stack traces for issues missing them (previously only at investigation time)
- Fetches time-series occurrence data per issue (`timeseries.json`)
- `Deps` interface for testable, decoupled engine design
- Adapter bridging engine to real Datadog client, reports manager, and SSE hub
- New Datadog API: `FetchErrorTimeline` using Spans Aggregate API with timeseries compute
- `GET /api/issues/:id/timeseries` endpoint for cached bucket data
- Config additions: `rate_limit` and `observation_window` in scan config

## What's Next

Five remaining feature slices build on this foundation:
1. Resolution Lifecycle — resolve in Datadog, monitor for regressions, confirm fixes
2. Notification Service — extensible notification abstraction with browser push
3. Dashboard Enrichment — sparklines, time window selector, trend indicators
4. Agent Investigation Enrichment — git correlation, frequency patterns, co-occurring errors
5. Config UI — settings page for managing config.yml from the browser
