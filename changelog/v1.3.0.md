# v1.3.0 — Single Binary, SSE & Browser Notifications

**Date:** 2026-03-28
**Design spec:**
- `docs/superpowers/specs/2026-03-28-single-binary-sse-notifications-design.md`

## Summary

Collapsed the multi-process architecture into a single binary serving the embedded frontend, API, and background scanner. Added real-time SSE event streaming and browser notifications for a fully live dashboard.

## Highlights

- Single binary: `./fido serve` runs API + embedded frontend + background scanner
- In-process SSE event hub with `GET /api/events` endpoint
- SSE-driven live dashboard updates with row highlights on changes
- Browser notifications on scan results and stage/CI changes
- Notification click navigates to issue detail
- Stream filter for real-time investigation progress
- Expanded dashboard rows with stack trace preview and Datadog link
- Fail-fast on startup if Datadog token is invalid
- Merged MR badge display instead of CI status
- Removed Docker Compose in favor of single binary
- Stripped AI thinking blocks from reports
