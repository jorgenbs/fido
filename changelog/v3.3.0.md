# v3.3.0 — API Key + Application Key Auth

**Date:** 2026-04-29

## Summary

Adds support for Datadog's API Key + Application Key authentication as an alternative to Personal Access Tokens (PATs), which aren't available on all Datadog plans.

## Highlights

- **Dual auth modes** — Configure either `token` (PAT with Bearer auth) or `api_key` + `app_key` (standard Datadog API auth). PAT takes precedence when both are set.
- **SDK-native API key auth** — API Key mode uses the official Datadog SDK's `ContextAPIKeys` mechanism, injecting `DD-API-KEY` and `DD-APPLICATION-KEY` headers automatically.
- **Config validation** — Clear error messages for incomplete auth (e.g. `api_key` without `app_key`), and a new `HasAuth()` helper on `DatadogConfig`.
- **Flat + multi-site support** — Both config formats accept the new `api_key`/`app_key` fields, and flat-format detection recognizes `api_key` as a format marker.
